Why

The 2026-06-09 autonomous run reported "557/557 features GREEN". A forensic critique found the claim inflated: 8% screenshot evidence, 86% of greens timestamped <2s apart (batch-marked), the fixer grading its own fixes, oracles derived from the code under test (circular), and a one-shot 20.7h run with no repeatable artifact. Graded: depth C-, evidence D, oracle C, repeatability D.

This effort hardened the system and proves the grades improved via a before/after forensic re-audit computed by a fixed, versioned tool (forensics.py) — not opinion.

What was built (Phases 1–4)

  1. Evidence gate — validate-state.py: a green requires screenshot|domAssert
    • oracleId + observed; write-paths additionally require dbAssert; a fixed feature requires verifiedBy ≠ fixedBy. push.sh refuses violating publishes.
  2. Oracle rebuild — 298 oracles re-derived from DESIGN.md / journey-map (provenance design), breaking the code-circularity of the baseline.
  3. DB assertion helpers — playwright/tests/perform-v2/helpers/dbAssert.ts: service-role read-backs on perform-dev (hard-guarded against staging/prod).
  4. Deterministic suite — playwright/tests/perform-v2/ (LANE=perform-v2, serial, mobile viewport): 241 specs across 13 areas, every write-path spec asserting UI and a DB read-back, red-route-b folded by reference. Runs green in ~9min (≤15min budget) — invocable via /perform-verify depth:'suite'.

Real production bugs the hardened suite caught

The DB read-back assertions found defects UI-only testing structurally cannot:

Bug Why UI-only missed it Fix
update_last_accessed_team RPC referenced nonexistent team_members.is_active → team-switch never persisted last_accessed_team localStorage masks it within a session migration 20260610130000 (status='active')

Plus dead-import removal (RoleSelectorDropdown) and an a11y fix (QuickViewSheet aria-labelledby pointed at a non-existent node).

Equally important — four cases where the test's own oracle was wrong, not the app (announcement-settings POST match, evals-outstanding RPC scope, notification mute table, hostedBy display-string), each caught by reproducing the mutation server-side before touching code. That two-way discipline is the gap the baseline had.
